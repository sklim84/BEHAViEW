## 🔍 변수 설명

- **md_type**: 거래 매체 구분 코드

  | 코드 | 설명       |
  |------|------------|
  | 1    | PC 뱅킹    |
  | 2    | 인터넷 뱅킹 |
  | 3    | 전화        |
  | 4    | 휴대전화    |
  | 5    | 건별이체    |
  | 6    | 기타        |
  | 7    | 대량이체    |

- **ff_sp_ai**: 사기 의심 유형

  | 값   | 설명                             |
  |------|----------------------------------|
  | 01   | 최근 3일 이내 대출 입금 후 송금 |
  | 02   | 예·적금 해지 당일 송금          |
  | SP   | 자체 기준에 따른 기타 사기 의심 |
  | NaN  | 사기 아님 또는 라벨 없음         |

## 📝 통합 해석 요약

- SP 사기거래는 모든 파일에서 주로 `md_type=4` (휴대전화)와 `md_type=2` (인터넷뱅킹)에 집중
- **HF_TRNS_TRAN 원본에서는 SP가 총 1,809건 중 1,685건이 `md_type=4`에 집중**
- **SP 기반 H1-hop 확장에서도 휴대전화(4번) → 234건, 인터넷뱅킹(2번) → 15건 탐지됨**
- 01/02 유형은 주로 HF_TRNS_TRAN 및 2025_03 원본에 포함되어 있으나 H1-hop에는 거의 포함되지 않음  
  → **SP 중심의 네트워크는 독립적 구조**를 가지며, **다른 유형과의 연계성은 약함**
